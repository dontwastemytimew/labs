# Лабораторна робота №3: Менеджер Нотаток (C++ / Qt)

Проєкт є застосунком для керування нотатками, який дозволяє користувачам динамічно створювати власні структури даних (схеми) та організовувати інформацію.

---

### Вимоги Лабораторної Роботи №3 (ООП)

Головною метою Лабораторної роботи №3 було **реалізувати програму, яка використовує зовнішні бібліотеки для реалізації частини функціональності** (крім стандартних бібліотек C++ та фреймворку Qt). Цей проєкт був значно доопрацьований та розширений для виконання цієї вимоги.

[Посилання на Звіт у Google Docs](https://docs.google.com/document/d/1AtfYBq3WI5Ve8d4pcb_eOiXYrrRThQtEziPmGtnJCyE/edit?usp=sharing)

### Стек Технологій та Зовнішні Бібліотеки

Проєкт використовує **C++20** та фреймворк **Qt** для кросплатформного GUI. Було інтегровано чотири ключові зовнішні бібліотеки:

| Бібліотека | Роль у Проєкті | Використаний Функціонал |
| :--- | :--- | :--- |
| **nlohmann/json** | *Основна Серіалізація.* Основна бібліотека для збереження/завантаження даних у формат JSON. | `json::parse`, `json::dump`, робота з `QMap`/`QList`. |
| **RapidJSON** | **Порівняльний Аналіз.** Використовується для порівняння швидкості I/O з `nlohmann/json`. | `rapidjson::Writer`, `rapidjson::StringBuffer`, порівняльний аналіз швидкості. |
| **nanobench.h** | **Бенчмаркінг Фреймворк.** Мінімалістичний фреймворк для точного вимірювання часу виконання функцій (наносекунди). | Точний вимір часу для порівняння JSON-бібліотек. |
| **QCustomPlot** | *Візуалізація Статистики.* Створення графіків (Гістограм) для відображення статистики використання програми. | Побудова графіків часу сесій у $StatisticsDialog$. |

### Ключові Доопрацювання та Функціонал

Проєкт був розширений наступними ключовими функціями та візуальними елементами:

-   **Система Організації:** Реалізовано повний функціонал **сортування** (за датою, назвою), **фільтрації** та **пошуку** по нотатках.
-   **Закріплення (Pinning):** Додано механізм **закріплення нотаток** для пріоритезації відображення.
-   **Вбудовані Медіа:** Можливість додавати **зображення** до нотаток.
-   **Мініатюри:** На плашках нотаток відображаються **мініатюри** (прев'ю) зображень та тегів.
-   **Експорт у PDF:** Додано можливість **експорту нотатки у формат PDF** для зручного друку.
-   **Статистика:** Збір та візуалізація статистики часу використання програми за днями за допомогою **QCustomPlot**.
-   **Візуальні Покращення:** Оновлено **дизайн** інтерфейсу (через QSS).

### Технічні Аспекти та Якість Коду

#### 1. Бенчмаркінг JSON I/O

Для об'єктивної оцінки продуктивності було реалізовано систему бенчмаркінгу, доступну у вікні налаштувань.

-   **Метод:** Вимірюється час серіалізації/десеріалізації $100,000$ синтетичних об'єктів для порівняння швидкості **`nlohmann/json`** та **`RapidJSON`**, використовуючи точний фреймворк **`nanobench`**.
-   **Результат:** Під час тестування **RapidJSON** показав себе в середньому **на 2.5–3 рази швидшим** за **nlohmann/json** у процесі запису.
-   **Висновок:** Бенчмарк підтвердив, що вибір бібліотеки залежить від пріоритетів: `nlohmann/json` зручніший для розробки, тоді як `RapidJSON` — для чистої швидкості.

#### 2. Юніт-Тестування

-   Проєкт містить **повний набір юніт-тестів** для забезпечення коректності та надійності.

#### 3. Документація, i18n та Логування

-   Проєкт повністю задокументовано з використанням **Doxygen**, документація генерується автоматично.
-   **i18n:** Повна підтримка перекладу інтерфейсу, включно з українською та англійською мовами.
-   **Логування:** Усі ключові події та помилки записуються у файл `log.txt` з часовими мітками.
